网络

同源策略 ⭐

所谓同源：协议、主机、端口号都相同的地址。

同源策略是指：当前页面在运行过程中发送的网络请求，必须时同源的，否则就会被认为是跨域请求。

浏览器中有一些不受同源策略限制的标签：script、img、link

所有可以通过script不受同源策略影响可以解决一部分跨域问题（jsonp）

同源策略是浏览器行为，为了保护本地数据不会被JS代码获取回来的数据污染，因此拦截的是客户端发出请求回来的数据接受，

即客户端发送了请求，服务器响应了数据，但是无法被浏览器接收

跨域解决办法 ⭐⭐⭐

jsonp

原理：准备一个自定义函数接收服务器数据，然后生成一个script标签，scr属性指向跨域站点，同时路径中需要添加准备的自定义函数作为回调函数

缺点：只支持GET请求

CORS跨域资源共享

原理：服务器设置Access-Control-Allow-OriginHTTP响应头之后，浏览器将会允许跨域请求。

请求时，客户端使用一些特殊的请求告知服务器跨域请求。服务器根据自身规则决定是否允许跨域，如果允许，则会通过响应头告知客户端可以发送跨域请求。

限制：浏览器需要支持HTML5、POST、PUT等，兼容IE9以上

nginx反向代理

nginx作为反向代理服务器，可以监听客户端的请求，并将这些请求转发到后端服务器。

通过将特定的前端项目域名映射到一个后端接口服务其上。

这样，当前的项目尝试访问后端接口时，nginx会检测到请求的前缀，并将请求转发到后端服务器。

后端服务器在处理请求时，不会知道这些请求实际上来自于nginx代理，因为nginx会通过rewrite命令去除掉请求中的前缀。

TCP 连接的过程

TCP协议通过三次握手建立可靠的对点连接

首先服务器进入监控状态，然后才可以建立连接

第一次握手：建立连接时，客户端发送SYN包到服务器，等待服务器确认。此次握手是客户端希望和服务器建立连接。

第二次握手：服务器收到SYN包后，然后回应客户端一个SYN+ACK包。此次握手服务器回应，表示已经收到并同意客户端连接请求。

第三次握手：客户端接受服务器的SYN包后，向服务器再发送ACK包。

最后服务器接收到ACK包，建立连接完成。

如何实现Token加密

常见的token格式JWT

token分为三段：header、payload、signature

分别标识：签名算法和令牌类型（header）、主体信息包含令牌过期时间，发布时间发行者，主体内容（payload）、特定的的算法对前面两部分加密，得到加密结果（signature）

在密钥不泄露的前提下，一个通过验证的token是值得信任的

get、post 的区别 ⭐⭐

两者的区别主要是浏览器默认行为导致的

  1. GET请求时，不会附带请求体。（GET请求不能传递data，POST可以传递params和data）
  2. GET请求传递的信息量有限，适合传递少量数据，POST请求传递信息量是没有限制的，适合传递大量数据。
  3. GET请求传递的params数据会附带到path参数中，能够通过分享地址完整的重现页面，但是也会暴露数据，如果有敏感数据不应该使用GET请求，至少不应该放到path中。
  4. 刷新页面时，如果当前页面是通过POST请求得到的，浏览器会提示用户是否重新提交；若是GET请求得到的页面则没有提示。
  5. GET请求的地址可以被保存为浏览器书签，POST不可以。
  6. GET请求只能传递ASCII数据，遇到非ASCII数据需要进行编码；POST请求没有限制。

cookie、Session Storage、Local Storage 的区别 ⭐⭐

都是保存本地数据的方式

cookie：兼容性好，所有浏览器都支持。

浏览器针对cookie会有一些默认行为，比如当响应头出现set-cookie字段时，浏览器会自动保存cookie的值；再比如，浏览器发送请求时，会附带匹配的cookie到请求头中。这些默认行为，使得cookie长期担任维持登陆状态的责任。与此同时，也正是因为浏览器默认行为，给了恶意攻击者可乘之机，CSRF攻击就是一个典型的利用cookie的攻击方式。虽然cookie不断改进，但前端仍然需要另一种更为安全的保存数据的方式。

HTML5新增了sessionStorage、localStorage。没有针对的浏览器默认行为。

sessionStorage：用于保存会话级的数据，浏览器关闭，不会保存数据

localStorage：用于更长久的数据保存，浏览器关闭重新打开，数据存在

cookie的大小有限制，一般浏览器会限制同一域下cookie总量为4M，而sessionStorage、localStorage没有大小限制。

cookie会与domain、path关联，而sessionStorage、localStorage只跟domain关联。

HTTP 请求的方法 ⭐

  1. GET：表示向服务器获取资源
  2. POST：表示向服务器提交信息，通常用于产生新的数据，例如注册，新增
  3. PUT：表示希望修改服务器的数据，通常用于修改
  4. DELETE：表示希望删除服务器的数据
  5. OPTION：发生在跨域的预检请求中，表示客户端向服务器申请跨域请求
  6. CONNECT：用于建立连接通道，通常在代理场景中使用，网页中很少用到
  7. TRACK：回显服务器收到的请求，主要用于测试和诊断
